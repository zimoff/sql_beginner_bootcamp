# Задания курса по SQL

## День 0  
В первый день курса будет рассмотрено, как работает реляционная модель (Relational Model) и как извлекать необходимые данные с использованием базовых конструкций SQL (Structured Query Language — язык структурированных запросов). 

**Примеры задач:**  
1. Создать таблицу с данными о клиентах (имя, фамилия, дата рождения, email).  
2. Написать базовый SELECT-запрос для извлечения всех записей из таблицы.  

**Используемые команды и функции SQL:**  
- SELECT  
- FROM  
- WHERE

---

## День 1  
Изучение методов получения данных на основе операций с множествами (sets constructions) и простых соединений (JOIN).

**Примеры задач:**  
1. Написать запрос для объединения данных из двух таблиц (например, клиенты и заказы).  
2. Использовать UNION для объединения результатов двух запросов.  

**Используемые команды и функции SQL:**  
- UNION  
- INTERSECT  
- EXCEPT  
- INNER JOIN

---

## День 2  
Практика работы с различными структурами соединений (JOINs) для извлечения данных.

**Примеры задач:**  
1. Использовать LEFT JOIN для получения списка всех клиентов и их заказов, включая тех, кто ничего не заказывал.  
2. Написать запрос с использованием CROSS JOIN для генерации всех возможных комбинаций двух наборов данных.  

**Используемые команды и функции SQL:**  
- INNER JOIN  
- LEFT JOIN  
- RIGHT JOIN  
- FULL OUTER JOIN  
- CROSS JOIN

---

## День 3  
Освоение изменения данных при помощи языка управления данными (DML — Data Manipulation Language).

**Примеры задач:**  
1. Написать запрос для добавления новой записи в таблицу (INSERT INTO).  
2. Обновить email клиента по его идентификатору (UPDATE).  
3. Удалить запись с данными о клиенте, который больше не активен (DELETE).  

**Используемые команды и функции SQL:**  
- INSERT INTO  
- UPDATE  
- DELETE

---

## День 4  
Изучение различий между виртуальными представлениями (View) и материализованными представлениями (Materialized View) в SQL.

**Примеры задач:**  
1. Создать обычное представление для отображения только активных клиентов.  
2. Создать материализованное представление для агрегации данных о продажах по месяцам.  

**Используемые команды и функции SQL:**  
- CREATE VIEW  
- CREATE MATERIALIZED VIEW  
- REFRESH MATERIALIZED VIEW  
- DROP VIEW, DROP MATERIALIZED VIEW

---

## День 5  
Обсуждение методов и принципов создания индексов для базы данных (indexes).

**Примеры задач:**  
1. Создать индекс на колонку с email для ускорения поиска.  
2. Проверить, как индекс влияет на скорость выполнения запроса.  

**Используемые команды и функции SQL:**  
- CREATE INDEX  
- DROP INDEX

---

## День 6  
Добавление новой бизнес-функции в существующую модель данных.

**Примеры задач:**  
1. Добавить новую таблицу для хранения информации о скидках.  
2. Обновить существующие запросы, чтобы учитывать эту таблицу.  

**Используемые команды и функции SQL:**  
- CREATE TABLE  
- ALTER TABLE  
- UPDATE

---

## День 7  
Применение агрегирующих функций для анализа и извлечения полезной информации из данных.

**Примеры задач:**  
1. Написать запрос для подсчёта общего количества заказов.  
2. Найти среднюю сумму заказа для каждого клиента.  
3. Определить максимальную и минимальную цену продукта в таблице.  

**Используемые команды и функции SQL:**  
- COUNT  
- AVG  
- SUM  
- MAX  
- MIN  
- GROUP BY

---

## День 8  
Разбор работы базы данных с транзакциями и уровнями изоляции (transactions and isolation levels).

**Примеры задач:**  
1. Написать транзакцию для выполнения нескольких связанных операций.  
2. Исследовать поведение базы данных при установке разных уровней изоляции (READ UNCOMMITTED, READ COMMITTED и т.д.).  

**Используемые команды и функции SQL:**  
- BEGIN TRANSACTION  
- COMMIT  
- ROLLBACK  
- SET TRANSACTION ISOLATION LEVEL

---

## День 9  
Создание и использование функциональных блоков (functional blocks) в базах данных.

**Примеры задач:**  
1. Написать пользовательскую функцию для расчёта налога.  
2. Создать хранимую процедуру для массового обновления записей.  

**Используемые команды и функции SQL:**  
- CREATE FUNCTION  
- CREATE PROCEDURE  
- CREATE TRIGGER

---

## День 10  
Реализация быстрого решения задачи Коммивояжера (Traveling Salesman Problem) с помощью SQL.

**Примеры задач:**  
1. Написать запрос для поиска кратчайшего маршрута между несколькими точками.  
2. Оптимизировать запрос для обработки большого объёма данных.  

**Используемые команды и функции SQL:**  
- WITH RECURSIVE  
- SELECT

---

## День 11  
Изучение концепции DWH (Data Warehouse — хранилище данных) и создание первого процесса ETL (Extract, Transform, Load — извлечение, преобразование, загрузка).

**Примеры задач:**  
1. Настроить процесс извлечения данных из оперативной базы.  
2. Создать ETL-скрипт для преобразования и загрузки данных в хранилище.  

**Используемые команды и функции SQL:**  
- INSERT  
- SELECT  
- UPDATE  
- DELETE
